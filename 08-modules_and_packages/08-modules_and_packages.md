# 08-modules_and_packages

## Learning goals

- Organize code using modules and packages.
- Use different import styles correctly.
- Understand `__init__.py` role in package structure.

## Key definitions

- `Module`: a Python file with code.
- `Package`: a directory of modules, usually with `__init__.py`.
- `Import`: loading names from module/package scope.

## Core concepts

- `import x` keeps a namespace prefix.
- `from x import y` imports specific names directly.
- Aliases (`as`) improve readability or avoid conflicts.
- Packages help scale code organization.

## Common mistakes

- Shadowing module names with local variable names.
- Running scripts from the wrong directory and breaking imports.

## Quick recap

- Good module/package layout improves reuse and maintainability.
